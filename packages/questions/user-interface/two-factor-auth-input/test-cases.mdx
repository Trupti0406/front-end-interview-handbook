## Test cases

- Invalid input should not be allowed in input boxes.
- Entering a valid input should fill the input box and move the focus to the next input box.
- Pressing Tab/Arrow buttons should move the focus to left/right based on the pressed key.
- Pressing Backspace should delete the current input (if filled) or move to previous input box and delete any input present from previous input box.
- Pasting a code should paste the code if the pasted code is valid input.
- Reset should be disabled if no input is present.
- Submit should be enabled only when all input boxed are filled with valid input.
- No interaction should be allowed when a network request is in progress.
